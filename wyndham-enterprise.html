Wyndham: Enterprise Platform

Designed the enterprise platform experience for use by customer service reps at the world's largest vacation timeshare company, with processes ranging from booking vacations for customers to managing membership accounts and executing administrative workflows.

( I also worked on the complementary consumer portal for use by the timeshare owners themselves. )

Problem

The challenge begins with timeshare owners having a difficult time with Wyndham Vacation Ownership's (WVO) consumer portal; this has increasingly pushed owners to dial in to the contact center for assistance. To add to that, contact center agents are currently using an archaic MS-DOS command-line system to handle all services, which not only is difficult to use (and look at) but also prolongs call times.

Thus, the client's ask was to deliver a revamped enterprise platform that promotes maximal efficiency for agents and reduced call times for customers while maintaining a consistent user experience with the consumer portal. This project was really engaging and intriguing since it not only required seamless design of the platform but also necessitated that we be cognizant of the human interaction between the agent and the calling customer.


Team

Myself, another UX designer, Director of UX, account supervisor, producer, business analyst, and 2 visual designers.


Process

We underwent a waterfall process for eight 4-week-long design sprints, one for each major section of the site. Each week I participated in a handful of informal pow-wows, 2-3 internal meetings, and 1-2 client calls during which the team collectively strategized, brainstormed, presented, and/or critiqued designs to ensure alignment with the client's requirements and timeline.

My tasks included:

  • Problem Deconstruction
    Analyzing functional requirements, all sorts of esoteric business rules, and high-level user needs.

  • Information Architecture
    Organizing, condensing, and hierarchizing data and requirements. Creating sitemaps and userflows.

  • Competitive Auditing
    Examining best practices and common design patterns for particular features (e.g., dashboards, results listings, data charts, forms).

  • Interaction Design
    Mapping out interaction models, process flows, and error studies. Translating IA into medium-fidelity desktop wireframes.

  • Functional Specifications
    Documenting technical and business requirements for page elements and the user interaction.

  • Client Presentation
    Structuring, creating, and annotating presentation decks. Walking through decks with the client.

"It takes too long to book a vacation over the phone."

In this process, agents are walking customers through the booking checkout flow. The main goal was to facilitate smooth communication between the agent and the customer while streamlining the agent interaction with the platform. I began this design sprint by mapping out the user flow shown below.

After finalizing this user flow with the client, we wireframed out the complete work flow. Below are a few notable steps.



VIP Upgrade

Some VIP Upgrade options may be available for eligible VIP customers, in which case the agent would communicate with the caller that they may opt to upgrade their desired unit.

Shown below is the max state scenario with the following options:

  1. Upgrade to a currently available VIP unit.
  2. Opt in for a VIP unit that may or may not become available later on.
  3. Do not upgrade altogether.

Charges

In the Charges step, the agent reviews all charges associated with the vacation booking. I arranged them in this layout in order to (1) visually distinguish Wyndham's different currency types, (2) hierarchically position them from most to least consumable, and (3) make complete use of the agents' allotted 1200 x 800 px viewport.

In the below scenario, the user does not have enough points to book the vacation, so the agent must prompt the user to designate a borrow and/or rent amount. I positioned these input fields just below their associated charge types.

"Admins are seasoned employees who know what they're doing."

This section of the platform is built specifically for a handful of agents with admin capabilities. They are well-trained individuals who have ownership over many system-wide tasks such as overriding policies/restricted actions as well as executing bulk functions (processes affecting numerous customers simultaneously).




Overrides

Admins have the ability to overrule certain restrictions if deemed acceptable and necessary while on-call with a customer. After receiving client-defined business requirements around override handling, I captured the logic in the below framework to determine how agents can proceed when an action is blocked (in this case, the agent is NOT "OK to Book"). Not only does the agent (or guide) have to be empowered to permit the override, but the customer must also be eligible for an override.


The below wireframes demonstrate the process through which an override is handled.

Here, the agent is met with an error indicating that an override is required in order to proceed.

Here, the agent is required to select a reason for the override before confirming the override and proceeding.

"The ability to navigate the interface and issue commands with keyboard shortcuts can greatly increase productivity and reduce call times."

Being a long-time enthusiast of hotkeys and productivity hacks, I volunteered to own this piece of the project. After exploring keystroke patterns from other platforms such as Gmail and feedly, I devised a keystroke framework that can be applied to a global navigation scheme that:

  • Is implementable across all sections of the platform.
  • Supports navigation
    • Between main columns/panels on screen.
    • Between content/functional groups.
    • Within content/functional groups.
  • Doesn't conflict with OS and browser standard keystrokes.
  • Supports global shortcuts for the most commonly used features.
